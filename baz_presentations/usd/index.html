What can USD do?

  • USD is a hierarchical data structure of Prims(Primitives) which contain Properties, both Prims and Attributes can have non-time varying metadata. These all live in a what is named a Layer which is alive on disk and/or memory
  • Theses Layers can then be composed into a greater scene description
  • You can build ontop of these basic prim types with your own custom Schema types. USD Comes with a bunch of schema types such as Xform, Mesh.
    • Reference

    • What can't USD do?

    • Can't override Prims by identifiers, only can target edits by "hierarchical namespace" identification
    • It isn't an evaluation engine(Animation evaluation engine, or like Houdini's execution graph), but you can make graphs that can be evaluated

    Terminology

    • Usd File anatomy --- USD Glossary
    • Stage
      • The object where all USD components live, such a Prim Prims
      • Provides access to the composed scene
    • Prims
      • Defines and Overs
      • Node in the USD hierarchy
      • Main container that encapsulates the following USD aspects
    • Sublayers
      • Pulls in scene description from a file on disk
      • The strongest of the composition-arcs

    More Anatomy of a USD file

    • References
      • Pulls in hierarchies from the specified reference
      • Main container that encapsulates the following USD aspects
      • Able to have external references and internal prim references
    • Payloads
      • Exactly like the reference, except it Lazy loads content
      • You can't pick and choose which payload is loaded, if you choose to load payloads you load all of them
    • Variant Sets
      • Allows you to dynamic swap poritions of scene description
      • Can only effect scene description below this point
    • Attributes and Metadata
    • Instancing
      • Is Implicit Instancing, where the master isn't explicitly defined but determined via a reference

    USD Workflows

    USD Allows you to:

    • Where you want a common interchange format between external people
    • Have different departments of people contributing to a piece of the scene(I don't mean in a realtime collaborative way)
    • Workflows where you want a department to non-destructivly affect the whole scene or part of it
    • Where you want to move away from any single DDC who owns the scene, but instead have many apps modifying a central model that is your scene

    Toolbox of a USD user

    • UsdCat
      • Prints out the contents of a USD file to stdout
      • Allows you to convert USD files between USD extensions or other extensions
        • >usdcat lexus_ux.usda
    • UsdView
      • Goto tool to test composition of your scene, test out variant switches and change values
        • >usdview lexus_ux.usda
    • UsdManager
      • Created by DreamWorks
      • Text editor based app that allows you to introspect compositon-arcs and follow them
    • AL_USDMaya
      • Create by Animal Logic
      • It's goal is round trip USD hierarchies in-and-out of Maya
