What can USD do?

  • It is photoshop for scene creation. Where you can put edits into Layers then compose the Layers to make a final image, or hierarchy in USD's case
  • Layers in USD are files and can then be `Composed` together into a greater scene description
  • Layers are made up of Prims who makeup a "hierachical namespace" or unique path to an object in your world
  • You use these hierarchical namespace paths to `Override` Prims in your scene. This allowes you to move trees and change the look of cars all by changing Attributes and Metadata on these Prims.
  • You can non-destructivly turn ON or OFF Prims and even Layers to modify your World.

    • What can't USD do?

    • Can't override Prims by identifiers or some UniqueID, only can target edits by this "hierarchical namespace"
    • It isn't an evaluation engine(Animation evaluation engine, or like Houdini's execution graph), but you can make graphs that can be evaluated

    Composition of a USD scene

    Terminology

    • Usd File anatomy --- USD Glossary
    • Stage
      • Composes Layers together
      • Provides access to the composed scene
    • Layers
      • Contains Prims(opinions)
      • Call pull in and compose other Layers via `SubLayers`
    • Prims
      • Are the nodes in a USD hierarchy
      • Can be specified as a Define(Def), Over or Class
      • Can have a Type and a Kind(i.e. Xform, Mesh)
      • Are the object with brings in the bulk of content into your scene

    More Anatomy of a USD file

    Intro to the Composition-Arcs and LiVRPs

    • Sublayers
      • Is Metadata of a Layer, not list editable
      • Pulls in scene description from a file on disk
      • The strongest of the Composition-Arcs
    • References
      • It's a list editable Metadata that lives on a Prim
      • Pulls in hierarchies from the specified file
      • Able to have external references and internal prim references
    • Payloads
      • Exactly like the reference, except it Lazy loads content
      • You can't pick and choose which payload is loaded, if you choose to load payloads you load all of them

    Even More Anatomy of a USD file

    • Properties, Attributes and Metadata
      • Attributes and Metadata are all Properties
      • Metadata is more lightweight than Attributes since they arn't affected by Time
      • There is all the standard Attribute types
      • More info: USDRelationship is powerful Property which can reference any part of the scene description(Intermediate)
    • Variant Sets
      • Allows you to dynamic swap poritions of your scene
      • Can only effect scene description below the point where the VariantSet lives
    • Instancing
      • Is Implicit Instancing, where the master isn't explicitly defined but determined by logic within USD that relies on a Reference

    USD Workflows

    USD Allows you to:

    • Where you want a common interchange format between external people
    • Have different departments of people contributing to a piece of the scene(I don't mean in a realtime collaborative way)
    • Workflows where you want a department to non-destructivly affect the whole scene or part of it
    • Where you want to move away from any single DDC who owns the scene, but instead have many apps modifying a central model that is your scene

    Toolbox of a USD user

    • UsdCat
      • Prints out the contents of a USD file to stdout
      • Allows you to convert USD files between USD extensions or other extensions
        • >usdcat lexus_ux.usda
    • UsdView
      • Goto tool to test composition of your scene, test out variant switches and change values
        • >usdview lexus_ux.usda
    • UsdManager
      • Created by DreamWorks
      • Text editor based app that allows you to introspect compositon-arcs and follow them
    • AL_USDMaya
      • Create by Animal Logic
      • It's goal is round trip USD hierarchies in-and-out of Maya
